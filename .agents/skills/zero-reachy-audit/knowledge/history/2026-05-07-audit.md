# Reachy Audit - 2026-05-07

## Trigger

Reliability repair after Live Conversation showed every subsystem green but produced zero transcript turns.

## Main Finding

Reachy was not failing at the daemon, Docker, body, or provider layer. It was failing at the audio signal layer: the Reachy Mini Audio input devices opened successfully, but they streamed digital silence. Zero was treating "microphone device opened" as "microphone ready", so the console looked healthy while no speech reached the realtime model.

## Fixes Landed

- Added actual input-signal classification to realtime sessions.
- Split frame arrival from signal arrival in input health.
- Added `last_frame_at`, `last_signal_at`, `confidence_state`, and `suggested_action` to session health.
- Marked prolonged Reachy mic digital silence as `reachy_mic_no_signal`.
- Added frontend automatic fallback to the browser/computer mic when Reachy mic signal is absent.
- Replaced generic "mic ready" UI with health-specific labels.
- Added OpenAI Realtime far-field noise reduction and explicit server VAD settings.
- Added regression tests for digital silence classification and fallback suggestion.
