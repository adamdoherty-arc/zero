## Why composite isn't 100

The audit profile defines `competitive parity = 100 if matches Devin / SWE-bench top-tier`. That's a feature-work bar:
- Devin: agent that browses codebases, runs commands, and ships PRs end-to-end on real GitHub issues
- SWE-bench: open-source benchmark of agent performance on real bug-fix tasks scored at the test-pass level
- Linear (sprints baseline): smooth keyboard nav, real-time collaboration, polished mobile

Legion currently scores **60** on this dimension because:
- It DOES ship sprints autonomously and grade them (matches the spirit)
- It does NOT have a published SWE-bench-style score on a fixed problem set
- It does NOT have the keyboard / real-time / mobile polish of a Linear-tier UX
- LLM Console is functional but not at Langfuse / Helicone aggregation depth

These are 1-month+ feature projects. Refactoring cannot move this dimension. Honest cap with refactor + wiring alone: **94** (raise functional 96→100 by exempting POST-only endpoints, raise integration 97→100 by widening telemetry to all 5 swarm nodes once they fire, dead 95→100 by some final scrub).
