## Cross-feature integration gaps

1. **Agent Dashboard ⇆ DB**: 40 agent definition files exist (`app/agents/*.py`), but `agents` and `agent_executions` tables are both empty. Page renders from in-code registry, not telemetry — no per-agent KPI, success rate, or cost over time.
2. **Project Grades roll-up**: `project_grades` = 0 rows; `plan_grades` = 61, `sprint_quality_grades` = 700. Aggregation pipeline plan→sprint→project is wired up to plans but never materializes the project tier. Known gap per CLAUDE.md.
3. **Product Docs ⇆ product_features**: page expects `product_features` rows; 0 in DB.
4. **Pull Requests ⇆ Sprints**: no `pull_requests` table; PRs come live from GitHub API and aren't persisted, so cross-feature joins (sprint → PR → release) cannot be queried — only fetched per-request from GitHub.
5. **`/projects/:id/dependencies` 422**: backend `/api/dependencies` requires `task_id`, but the page passes `project_id`. Either route needs a project-scoped variant or the page hook is broken.
