# Discord Formatting Rules

When formatting responses for Discord delivery, follow these rules:

## Do
- Use **bold** for emphasis (double asterisks)
- Keep responses to 1-3 sentences when possible
- Use bullet points for lists
- Use `code` backticks for technical values, IDs, commands
- Use emoji sparingly for tone (not decoration)
- Split long responses at natural breakpoints (paragraphs)

## Don't
- No `## headers` — Discord renders them poorly. Use **bold text** instead.
- No markdown tables — use bullet lists for structured data.
- No horizontal rules (`---`)
- No blockquotes unless quoting someone
- Don't exceed 1500 characters per message chunk (Discord limit is 2000, leave buffer)

## Formatting Data
When presenting sprint/task/email data:
- **S1787** — Zero TikTok Shop Automation (14 tasks, 8 done)
- Use bold for identifiers, dash for separator, parentheses for details
- One item per line for readability
