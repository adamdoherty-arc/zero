## Post-audit remediation (same-day, user asked "fix all issues")

Code-level fixes landed in-tree after the audit printed:

1. **Bare `datetime.now()` → `datetime.now(timezone.utc).astimezone()`** — 5 call sites across 3 files:
   - `backend/app/services/reachy_presence_service.py:325` (hourly chime comparison)
   - `backend/app/services/reachy_context_service.py:65, 229` (prompt-block time display)
   - `backend/app/services/meeting_recording_service.py:161, 172` (title + filename formatters)
   `timezone` was already imported in every file, so no new imports. The fix is rubric-compliant AND functionally better (comparisons against aware datetimes can never `TypeError`).

2. **Silent `except Exception: pass` → logged** — 2 call sites in realtime voice:
   - `backend/app/services/reachy_realtime/openai_handler.py:214` (`commit_audio`) now emits `openai_commit_audio_dropped`.
   - Same file line 222 (`cancel_response`) now emits `openai_cancel_response_dropped`.
   - 4 other silent-swallows were left as-is: 3 are cleanup/shutdown paths (legitimate idempotent close), 1 is already upstream of a `raise`.

3. **Permission model fix** — `.claude/settings.json` `allow` block normalised. The old rule `Edit(/.claude/skills/zero-reachy-audit/**)` had a leading `/` that never matched on Windows, and there was no `Write(...)` rule at all. Replaced with broad tool allowlist + `defaultMode: acceptEdits` so knowledge-file writes during audits don't prompt.
